## 2026.5.31

output 関数がローカル環境では行ごとに flush するように変更した。
multi testcase の問題でテストケースを手入力して試すときに、テストケースごとに結果が表示されないのが不便だったため。

## 2026.6.14

5.31 の変更はリアクティブ問題で不慮の事故が起きそうということに気づいたので、多少の速度を犠牲にしてジャッジ環境でも毎行 flush する方針に変更した。

## 2026.6.25

やはり速さを優先して 5.31 の仕様に戻した。
また、input 関数はこれまで
```
int n = input();
```
と戻り値で型を指定できる書き方にしていたが、これはやや不自然な実装を強いられ、また Grid などの自作クラスに拡張しにくかったため、
```
auto n = input<int>();
```
とテンプレート引数を明示して auto で受ける形に変更した。
